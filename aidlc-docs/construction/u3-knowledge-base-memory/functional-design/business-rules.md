# Business Rules
# U3 ‚Äî Knowledge Base & Memory System

**Date**: 2026-04-11

---

## KnowledgeBase Rules

### KB-01 ‚Äî Corrections Log is Append-Only
- `kb/corrections/corrections.json` MUST NEVER be truncated or overwritten with fewer entries than it previously contained.
- `append_correction` always reads the existing list, appends, and writes back the full array.
- Source: FR-08

### KB-02 ‚Äî CHANGELOG.md Files are Append-Only
- Each subdir's `CHANGELOG.md` is opened in append mode (`"a"`) ‚Äî never `"w"`.
- `inject_document` appends an ISO-8601 timestamped entry with filename and token estimate.
- The CHANGELOG is never included in `load_documents` results (filtered by name check).

### KB-03 ‚Äî Session Transcripts are Write-Once
- `save_session` checks if `sessions/{session_id}.json` already exists.
- If it exists, the call is a no-op (logs a warning) ‚Äî never overwrites.
- No method in U3 deletes or modifies an existing session transcript.

### KB-04 ‚Äî Document Cache TTL per Subdir
- Each subdir has its own independent cache entry `(docs, loaded_at)`.
- Cache is invalidated when `inject_document` adds a new document to that subdir.
- Refresh interval is read from `settings.layer2_refresh_interval_s` (default: 60s).
- An empty subdir returns `[]` ‚Äî not an error.

### KB-05 ‚Äî Injection Test: 4,000 Token Budget
- Before writing any document, `inject_document` estimates token count as `len(content) // 4`.
- If the estimate exceeds 4,000, `ValueError` is raised ‚Äî the file is NOT written.
- The document content must be valid UTF-8 (encode check performed before write).
- Source: Q2=B

### KB-06 ‚Äî Atomic Corrections Write
- `append_correction` writes to `corrections.json.tmp` first, then calls `Path.replace()` (atomic rename).
- This prevents corruption from concurrent writes or process interruption mid-write.
- On Windows, `replace()` is best-effort atomic; a failed rename leaves the `.tmp` file in place (not the original).
- Source: Q3=C

### KB-07 ‚Äî KB Directory Auto-Init
- `_ensure_kb_structure()` creates all 4 subdirs and their CHANGELOG.md files on first startup.
- Uses `mkdir(parents=True, exist_ok=True)` ‚Äî idempotent and never raises on existing dirs.
- CHANGELOG.md is only seeded if it does not already exist (never overwrites existing content).
- Source: Q9=D

### KB-08 ‚Äî In-Memory Corrections Mirror
- `_corrections` list is always the authoritative in-memory mirror of what is on disk.
- `append_correction` updates both disk (atomic) and `self._corrections` in the same call.
- `get_corrections()` returns a defensive copy ‚Äî callers cannot mutate the internal list.
- Source: Q4=A

### KB-09 ‚Äî CHANGELOG Not Served to LLM
- `load_documents` explicitly skips any file named `CHANGELOG.md`.
- CHANGELOG is an operational artifact for developers ‚Äî not KB content for context injection.

### KB-10 ‚Äî Subdir Validation
- `load_documents` and `inject_document` only accept subdirs in `{"architecture", "domain", "evaluation", "corrections"}`.
- An unknown subdir raises `ValueError` immediately ‚Äî no partial I/O occurs.

## MemoryManager Rules

### MM-01 ‚Äî Session Files are Write-Once
- `save_session` checks existence before writing.
- Duplicate `session_id` ‚Üí log warning, return immediately without touching the file.
- Source: KB-03

### MM-02 ‚Äî autoDream Triggers at Startup
- `MemoryManager.__init__` schedules `_run_autoDream()` as an asyncio background task.
- autoDream only processes sessions older than `memory_max_age_days` (default: 7 days).
- Sessions younger than the threshold are left untouched.
- Source: Q7=B, NFR-03

### MM-03 ‚Äî Consolidation is Additive Only
- `_merge_session_into_topics` NEVER removes entries from topic files.
- Deduplication is by `session_id` ‚Äî already-consolidated sessions are skipped.
- `user_preferences` merge uses `dict.update()` ‚Äî newer values overwrite on key collision.
- Source: Q8=C

### MM-04 ‚Äî Topic Files Written Atomically
- `_write_topics_atomic` uses the same `.tmp` ‚Üí `replace()` pattern as KB-06.
- All three topic files are written in sequence (not parallel) to avoid partial update state.

### MM-05 ‚Äî MEMORY.md is Append-Only
- `_run_autoDream` opens `MEMORY.md` in append mode when writing the consolidation summary block.
- The MEMORY.md index is never truncated or rewritten from scratch after initial creation.

### MM-06 ‚Äî Session Transcript Contains Both Raw and Summary
- `save_session(session_id, history, summary)` requires both arguments.
- `history: list[TraceStep]` is the full ReAct trace ‚Äî never abbreviated before persistence.
- `summary: str` is a caller-provided text description (generated by U1 Orchestrator).
- Source: Q6=C

### MM-07 ‚Äî No Network Dependencies
- MemoryManager performs file I/O only ‚Äî no HTTP calls, no LLM calls.
- Summary text is passed in by the caller (U1 Orchestrator) ‚Äî MemoryManager never generates it.
- This keeps U3 fully testable without MCP Toolbox or OpenRouter.

### MM-08 ‚Äî Memory Directory Auto-Init
- `sessions/`, `topics/`, and `MEMORY.md` are created if missing in `__init__`.
- Never raises `FileNotFoundError` on missing memory directory at agent startup.

### MM-09 ‚Äî No PII in File Names
- Session files are named `{session_id}.json` ‚Äî session_id is a UUID, not user-identifiable.
- No query text, user data, or PII appears in file names anywhere in the memory system.

## Data Contract Rules

### DC-01 ‚Äî CorrectionEntry Model Compatibility
- All entries in `corrections.json` must deserialize to `CorrectionEntry` without error.
- On `_load_corrections_from_disk`, any entry that fails to parse is logged and skipped ‚Äî not raised.
- This prevents one corrupted entry from blocking the entire corrections load.

### DC-02 ‚Äî SessionTranscript Structure
- Sessions are serialized with `model_dump_json()` (Pydantic) ‚Äî guaranteed round-trip safe.
- `history` field is a list of `TraceStep` dicts; loaded back with `TraceStep(**item)` unpacking.

### DC-03 ‚Äî Topic Files are Valid JSON
- `_write_topics_atomic` writes with `json.dumps(..., default=str)` ‚Äî non-serializable values are coerced to strings rather than crashing.
- On load, missing topic files return empty `[]` or `{}` rather than raising.
