# Claude Code — autoDream Memory Consolidation

autoDream is a memory consolidation mechanism that transforms repeated patterns and experiences into structured knowledge.

## Function

Instead of storing all interactions verbatim, the system:
- identifies recurring patterns
- extracts stable insights
- promotes them into persistent memory

## Process

1. Observe repeated interactions or failures
2. Identify consistent patterns or fixes
3. Convert into structured knowledge entries
4. Store in topic-level memory

## Benefits

- Reduces redundancy in memory
- Improves efficiency of future reasoning
- Enables system-level learning without retraining

## Application to Oracle Forge

autoDream maps directly to the **KB corrections layer (v3)**:

- failed query → diagnosis → correct pattern
- repeated failures → standardized fix
- fixes become reusable knowledge

This enables:
- self-improving behavior
- reduced repeated errors
- stronger evaluation performance over time

The key idea is:
**learning is achieved through memory restructuring, not model updates.**
