# Scoring Methodology

OracleForge uses pass@1 semantics aligned to DAB workflow:

- Group trials by `(dataset, query_id)`.
- A query "passes" if any trial passes.
- `pass@1 = passed_queries / total_queries`.

Artifacts generated by `eval/score_results.py`:
- `dab_detailed.json` (per-query diagnostics)
- `dab_submission.json` (submission summary)

Evaluation quality checks:
- Score must remain in `[0.0, 1.0]`.
- At least two score snapshots should be kept to show progression.
- Runtime trace IDs and tool-call evidence should be retained for failed cases.
